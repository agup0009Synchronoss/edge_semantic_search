# Edge Semantic Search

A Android POC app that searches your photo library using natural language вЂ” no server, no cloud, no account required. [EDGE DEVICE SEMANTIC SEARCH] 

Type **"dinner with friends"** or **"sunset at the beach"** and the app ranks your photos by semantic similarity in milliseconds. Everything runs on-device: the ML models, the tokenizer, the index, and the search.

---

## What it does

| Stage | What happens |
|---|---|
| **Background indexing** | WorkManager job (charging + idle) scans new photos via MediaStore, runs each through the vision encoder, and stores the 512-d embedding in a Room database |
| **Live query** | You type a query; the text encoder embeds it in real time |
| **Ranking** | Brute-force cosine similarity across all stored embeddings; top results shown in a ranked grid |

No network calls are made at any point.

---

## Benchmark UI

The app ships a benchmark screen that lets you measure performance end-to-end:

1. **Pick N images** вЂ” select any photos from your gallery
2. **Embed** вЂ” runs the vision encoder on each; shows a live log with per-image timings:
   - `Decode ms` вЂ” BitmapFactory decode time
   - `Infer ms` вЂ” ONNX Runtime inference time
   - `Total ms` вЂ” wall time per image
3. **Search** вЂ” type a text query; shows text embedding latency in ms
4. **Ranked results** вЂ” images sorted by cosine similarity, score shown to 3 decimal places (e.g. `cos: 0.284`)

## Architecture

```
MediaStore
    в””в”Ђв–є IndexWorker (WorkManager, charging+idle)
            в””в”Ђв–є ClipEngine.encodeImage()   vision_model_fp32.onnx
                    в””в”Ђв–є FloatArray[512] L2-normalized
                            в””в”Ђв–є Room DB (mediaId, embedding blob)

Query text
    в””в”Ђв–є ClipEngine.encodeText()   text_model_int8.onnx + custom_op_cliptok.onnx
            в””в”Ђв–є FloatArray[512] L2-normalized
                    в””в”Ђв–є dot product with all stored embeddings
                            в””в”Ђв–є ranked URIs в†’ Coil image grid
```

Key files:

| File | Role |
|---|---|
| `EdgeSearchApp.kt` | Application class; copies assets, builds ORT sessions, exposes `engineReady` StateFlow |
| `ml/ClipEngine.kt` | Owns two ORT sessions (vision fp32, text int8) + tokenizer session; `encodeImage()` / `encodeText()` |
| `ml/ImagePreprocess.kt` | Bitmap в†’ resize/crop 224Г—224 в†’ normalize в†’ NCHW float32 tensor |
| `ml/Tokenizer.kt` | Wraps `custom_op_cliptok.onnx`; pads/truncates to seq_len=77; BOS=49406, EOT=49407 |
| `data/AppDatabase.kt` | Room DB вЂ” entity: `(mediaId, dateAdded, embedding: ByteArray)` |
| `index/IndexWorker.kt` | CoroutineWorker; incremental (skips already-indexed media IDs) |
| `index/IndexRepository.kt` | MediaStore diff в†’ encode new photos в†’ batch insert |
| `search/SearchViewModel.kt` | `BenchmarkUiState`, live embed log, cosine ranking |
| `search/SearchScreen.kt` | Jetpack Compose UI вЂ” picker, embed log card, search bar, result grid |
| `tools/export_split.py` | One-time dev-machine script to produce the ONNX assets |

## ML Models

### Model: TinyCLIP-ViT-8M-16-Text-3M-YFCC15M

A compact CLIP model from the [TinyCLIP paper](https://arxiv.org/abs/2309.04504), projection dimension 512.

| Asset | Source | Size | Notes |
|---|---|---|---|
| `vision_model_fp32.onnx` | PyTorch export of `CLIPVisionModelWithProjection` | ~33 MB | fp32; int8 variant unusable on ORT Android (no `ConvInteger` kernel on CPU EP) |
| `text_model_int8.onnx` | Extracted from HF pre-quantized combined model via `onnx.utils.extract_model` | ~15 MB | int8 |
| `custom_op_cliptok.onnx` | Generated by `onnxruntime-extensions` `SingleOpGraph.build_graph(CLIPTokenizer)` | ~1.4 MB | CLIP BPE vocab + merges embedded as attributes |

**HuggingFace model repo used:**
[`onnx-community/TinyCLIP-ViT-8M-16-Text-3M-YFCC15M-ONNX`](https://huggingface.co/onnx-community/TinyCLIP-ViT-8M-16-Text-3M-YFCC15M-ONNX)
(original: [`wkcn/TinyCLIP-ViT-8M-16-Text-3M-YFCC15M`](https://huggingface.co/wkcn/TinyCLIP-ViT-8M-16-Text-3M-YFCC15M))

### Runtime

- **ONNX Runtime Android** `1.20.0` вЂ” [`com.microsoft.onnxruntime:onnxruntime-android`](https://mvnrepository.com/artifact/com.microsoft.onnxruntime/onnxruntime-android)
- **ONNX Runtime Extensions Android** `0.13.0` вЂ” [`com.microsoft.onnxruntime:onnxruntime-extensions-android`](https://mvnrepository.com/artifact/com.microsoft.onnxruntime/onnxruntime-extensions-android) (provides the `CLIPTokenizer` custom op)
- NNAPI EP enabled with CPU fallback

## Producing the ONNX assets (one-time, dev machine)

The three `.onnx` files are **not committed** (too large for Git). Run the export script once to produce them and copy them into `app/src/main/assets/`:

```bash
# Python 3.9+ required
pip install onnx onnxruntime onnxruntime-extensions==0.13.0 huggingface_hub

python tools/export_split.py
```

The script:
1. Downloads `model_int8.onnx` from HuggingFace (cached after first run)
2. Extracts the text subgraph в†’ `text_model_int8.onnx`
3. Builds the CLIPTokenizer ONNX with vocab+merges embedded в†’ `custom_op_cliptok.onnx`
4. Validates both models (smoke-test embeddings + tokenizer token IDs)
5. Copies all three files to `app/src/main/assets/`

> **Note:** `vision_model_fp32.onnx` must be exported separately using PyTorch + `transformers`:
> ```python
> from transformers import CLIPVisionModelWithProjection
> import torch
> model = CLIPVisionModelWithProjection.from_pretrained("wkcn/TinyCLIP-ViT-8M-16-Text-3M-YFCC15M")
> dummy = torch.zeros(1, 3, 224, 224)
> torch.onnx.export(model, dummy, "vision_model_fp32.onnx",
>                   input_names=["pixel_values"], output_names=["image_embeds"],
>                   opset_version=14)
> # then copy to app/src/main/assets/
> ```
> The int8-quantized vision model cannot be used on ORT Android because the `ConvInteger` kernel
> (used by dynamically-quantized patch embedding `Conv`) is not implemented in the Android CPU EP.

## Build & install

Requirements: Android Studio Hedgehog or later, JDK 17+, Android SDK 35, device/emulator API 26+.

```bash
# Debug build
./gradlew assembleDebug

# Install on connected device
adb install -r app/build/outputs/apk/debug/app-debug.apk

# Or build + install in one step
./gradlew installDebug

# Launch via ADB
adb shell am start -n com.edgesearch.app/.MainActivity

# Watch app logs
adb logcat -s EdgeSearchApp ClipEngine Tokenizer
```

On Windows, use `gradlew.bat` and ensure `JAVA_HOME` points to the Android Studio JBR:

```bat
set JAVA_HOME=C:\Program Files\Android\Android Studio\jbr
gradlew.bat assembleDebug
```
